## Chrome AI 网页爬虫插件需求文档 (PRD)

### 1. 产品定位

一款基于 AI 视觉与 DOM 分析的智能爬虫工具。通过 AI 自动识别网页结构，降低零代码用户的配置门槛，支持复杂分页与自动化调度。

### 2. 核心功能模块

#### 2.1 AI 智能解析引擎 (AI Selector)

* **配置管理：** 支持标准的 OpenAI API 格式（Base URL, API Key, Model Name）。
* **交互选取：** 用户开启“选择模式”后，悬停高亮 DOM 元素。
* **语义化提取：**
* 插件捕获选中元素的 HTML/文本片段。
* **Prompt 逻辑：** 插件自动生成提示词（例如：“请分析这段 HTML，提取出所有重复的列表数据字段，并以 JSON 格式输出字段名和示例值”）。
* **结果确认：** 界面展示 AI 提取的字段名（如：价格、标题、描述），用户通过复选框决定最终抓取的列。



#### 2.2 数据采集与分页逻辑 (Scraping & Pagination)

* **自动分页识别：** AI 尝试识别“下一页”按钮或滚动加载逻辑。
* **手动定义分页：** 用户可手动指定“翻页按钮”或定义 URL 规则（如 `?page={n}`）。
* **混合采集模式：**
* **AJAX/SPA：** 模拟点击翻页，监控 DOM 变化。
* **URL 跳转：** 识别 URL 参数，支持页面重载后的脚本自动续接。


* **去重机制：** 基于数据指纹（Hash）防止翻页过程中的数据重复。

#### 2.3 任务持久化与断点续传 (Persistence)

* **本地存储：** 使用 `chrome.storage.local` 或 IndexedDB 存储任务状态。
* **状态机模型：** 记录当前任务的 `Status` (运行中/暂停/完成)、`Current_Page`、`Config`。
* **热启动：** 浏览器重启后，检测到未完成任务，弹窗提示“是否恢复上次爬取”。

#### 2.4 数据管理与导出

* **实时预览：** 采集过程中，界面实时滚动展示最新获取的 5 条数据。
* **格式支持：** 首期支持 CSV，二期可规划 JSON, Excel。
* **存储路径：** 支持调用 Chrome Download API 直接下载。

---

### 3. 用户界面 (UI/UX) 设计要求

* **侧边栏布局 (Side Panel)：** 推荐使用 Chrome Side Panel API，避免遮挡网页主体内容。
* **步骤条引导：**
1. AI 配置 -> 2. 元素选取 -> 3. 字段确认 -> 4. 运行采集。


* **可视化反馈：** 正在采集时，页面被选中的元素应有动态边框流光效果。

---

### 4. 异常处理机制 (Error Handling)

| 错误类型 | 处理策略 |
| --- | --- |
| **AI 接口超时** | 自动重试 3 次，若失败则提示用户检查 API Key 或网络。 |
| **分页元素消失** | 停止抓取，并保留已获取数据，提示“未找到翻页元素”。 |
| **反爬虫拦截** | 监测到重定向至验证码页面时，暂停任务并弹出系统通知提醒用户手动干预。 |
| **存储溢出** | 监测浏览器存储空间，必要时分批导出。 |

---

### 5. 自动化任务调度 (Automation)

* **无头运行限制：** 需明确告知用户，由于 Chrome 插件限制，定时任务通常需浏览器处于开启状态。
* **配置项：**
* 执行频率（每小时、每天、每周）。
* 静默执行：在后台 Tab 页自动打开目标 URL 并按预设配置抓取。
* 成功通知：任务完成后发送系统通知或邮件（需后端支持）。



---

### 6. 技术实现建议 (Tech Stack)

* **核心框架：** React/Vue3 + Tailwind CSS (UI)。
* **通信机制：** `Content Script` 负责 DOM 操作，`Background Service Worker` 负责 AI 请求与数据持久化。
* **AI 交互：** 使用 `LangChain.js` 或直接调用 `fetch` 适配 OpenAI 协议。

### 7. 优化建议 (给你的思考)

1. **关于 AI 费用：** 每次点击解析都会消耗 Token。建议增加“缓存模板”功能。如果用户在同一网站再次抓取，直接复用上次的解析规则，无需再次请求 AI。
2. **数据清洗：** AI 提取的数据可能带有空格或 HTML 标签，建议在导出前增加一个“简单清洗”步骤（去空格、正则过滤）。
3. **速率限制：** 自动翻页太快容易被封 IP，建议在分页设置里允许用户设置“翻页延迟时间（1-5秒）”。
